    指标

    我怎么看顶会的CB-NTR的预测代码中, 也是将logit和label直接送入评价指标中计算呢.

    多标签阈值

    我发现, 无论是法律多分类, 知乎多分类, 还是事件抽取任务多分类都采用大于0.5的方式来预测, 但是这时候就有问题了…就是样本中, 可能某个标签所含的样例很多, 一些标签所含的样例很少, 比如有

    [1, 0, 1, 0, 0]
[1, 1, 0, 0, 0]
[1, 0, 0, 1, 0]
[1, 0, 0, 0, 1]

    这个样例就是index=0这个标签出现的样例太多了, 所以这里可能就出现问题, 在预测的时候index=0的位置的概率总是很大, 这里就需要评估指标进行量化. 也就是存在数据不均衡的问题.

    那么0.5用来判断合适不合适?, 不一定合适, 因为也许对于index=0的时候, 0.8更合适, 这里的0.8就可以统计训练数据中1占所有数据中的比例来做阈值!

    实验

    分析, 因为样例中信息信息安全室这个标签有很多样例, 但其它标签的样例都很少, 所以(假设第一个位置为信息安全室标签)样本在拟合的时候, 更容易把稀疏标签更容易拟合成0, 也就是以更小的概率接近为1, 甚至如果样本中, 信息安全室标签的样本小于非信息安全室标签的样本的数量的时候, 模型更容易预测一个位置为非信息安全室, 也就是概率就会小于0.5.

    思考

    1. 多标签任务中, 也许有的样例只有一个标签, 可以对多标签和单标签进行统计, 也许可以从这个角度进行优化.
    2. 那么0.5用来判断合适不合适?, 不一定合适, 因为也许对于index=0的时候, 0.8更合适, 这里的0.8就可以统计训练数据中1占所有数据中的比例来做阈值!
    3. 从感觉来说, 预测标签打多的话比打少更好一点, 打多了只是信息冗余, 让多的那个标签来处理一下即可, 打少了的话是信息缺失,后果可能更严重.
    4. 文本可不可以用dropout
    5. 有的多标签分类任务和有的多标签分类任务是不同的, 比如一个电影的多标签分类, 是多标签ranking, 也许只取Top3的标签就可以了, 也可以用threshold来区分, 这种标签的数目准确与否是无所谓的, 但是如果是我想解决的邮件正文多标签分类, 如果分错了, 也许一些邮件内容需要部门进行内容隔离的, 就比较注重标签的准确性.
    6. threshold的设置到底取决于什么因素, 用0.5到底有什么问题, 是不是和样本的不均衡性有关系, 可否用某个标签占所有样本中的比例来做threshold.

    1. 看各项指标代表什么意思, 多少才算号.
    2. 增大epoch, 看什么时候收敛
    3. 改变threshold, 看哪个threshold可以对样例得出不同的输出.
