    知识体系

    背景

    什么是学习?

    上了大学很是迷惑，面临新的科目时，只给一门科目名，其他的都需要自己去寻找，学习资料，学习方法，考核标准，实践标准等，不同于中学的老师给你所有的安排妥当，大学只能自己探索，所以自然而然的思考：

    什么是学习？什么是读书？读书的结果是什么？

    去看了一些书，比如《如何阅读一本书》，《怎样读书》，《读书的艺术》，《我的苦学经验》等方法论的书籍，做了读书笔记笔记，但尝试些许，并没有成功，最关键的原因是：

    1. 坚持不下来。
    2. 打破自己已往的学习习惯，形成新的学习习惯很难。
    3. 没有搞明白知识体系是什么东西。
    人的自身如何塑造?

    于是，转而又思考其他问题：

    人存在的形式是什么？我为什么做不到以上几点，自己何以为自己？自控力怎样获得？记忆力又如何加强？为什么我会在看书时思前想后，注意力又是怎样的一门科学？

    然后又去读了一些书，比如《高效能人士的实践和个人管理法则》，《提高你的记忆力》，《注意力曲线》等等，还找了知乎上的有些回答来看。

    什么叫做会一门技能?

    还有自己的专业，如何评定一个人是否会C++？他学习好的证据是什么？当时他学了《计算机网络》时成绩很高，但是现在一问他知识，他说他忘了。

    那他到底算不算会计算机网络？

    结论

    最终的结论是：

    1. 知识的本质就是记忆，而知识体系的构建可以帮助回忆和记忆。
    2. 人的本质也是记忆，人的各项能力可以自我塑造，但是依然建立在记忆的基础之上，你必须要不断重复自己的目标，自己所处的方位，必须摒弃现实世界混乱信息的干扰，要不忘初心。
    3. 到底会不会某个技能，这个标准也是在你的知识体系下的一个结果。

    所以，我必须建立起自己的知识体系，别人的形式如下

    而我必须在长时间的坚持下形成自己的知识体系。

    知识体系划分

    知识密度

    按照知识密度不同，常用的知识和资料分类如下：

    1. typora笔记和zotero论文
      知识最终是要成为自己的东西，所以必须要逼迫自己输出出来并且经常回忆，否则都是符号而已。
    2. 学习资料, 项目资料, 代码, 网页链接.
      这一部分承载了一些知识, 但是知识并没有内化, 只能开箱即用, 借鉴式的使用.
    3. 环境, 软件, 生活资料, 思维导图
    4. 豆瓣, B站, 知乎等使用频度较高的App
      结合网页链接和自身收藏夹使用.

    核心就是，要经常整理，然后输出，否则看了相当于没看。

    博客搭建

    第三方博客平台

    优点: 没有技术门槛

    缺点:

    1. 网站的内容可能因为网站本身被丢失, 比如博客园整改 gitee人工审核.
    2. 可定制化方案少, 无法按照自己的知识体系进行梳理.

    自己搭建博客例如wordpress

    优点:

    1. 带有后台的网站, 可以管理文章, 评论, 相册, 资源, 用户等.
    2. 可以在线编辑文章和内容.

    缺点:

    1. 需要自己购买域名和服务器.
    2. wordpress采用的文本编辑器是谷腾堡, 如果脱离了wordpress内容很难迁移.
    3. 不能在本地进行创作(可以在本地进行创作后再复制粘贴到网站)
    4. 服务器的部署和维护可能会占用一定精力, 不能专注于内容.
    5. 可迁移性差.

    静态博客

    优点:

    1. 可迁移性高, 可以本地创作, 只是将静态博客作为一个展示窗口, 耦合性低.
    2. 技术难度小(相比于自己建站).
    3. 可以高度定制化
    4. 专注于写作

    缺点:

    1. 静态博客想要实现交互(评论)功能时, 不太方便, 需要使用第三方接口, 关于静态博客的评论系统介绍了一些解决方案.

    代码管理

    调试小项目我用 VS Code，大项目上 IDEA 系的软件，看文件用‘ Sublime Text ’。

    管理需求

    1. 别人的代码项目感兴趣但不一定有时间看的
    2. 别人的代码需要运行的
    3. 别人的代码需要改动并且添加功能的
    4. 自己写的代码片段, 不足以成为项目的
    5. 自己写的代码项目

    管理办法

    2022/08/05

    将代码分为两大部分: git 和 mycode, 代码所需要的数据放在data文件夹下

    • git
      • python
        • project1
        • project2
      • C++
      • ….
    • mycode
      • 1-code-snippets
      • project1
        如果1个project有比如前后端两个部分的, 把前端和后端视为两个project进行版本管理, 对于总project用submodule来管理
      • …
    • data
      • project

    git:

    先进行语言分类, 再进行project分类, 如果含兴趣自然star, 如果改动很多, 则将project整个文件夹移动到mycode 中进行自己project的建立, 如果只是学习借鉴, 则放在这个文件夹, 并且要经常多文件夹做整理.

    mycode:

    在code-snippets中以语言分类, 在语言中又以整个项目为单位做版本管理, 本科时候的代码片段则集合在一个project中, 比较大的代码量的话则作为一个project放在根目录.

    在更目录下面的话都是project, 信息量比较大.

    data:

    把所有需要用到的数据集按照project来放在这里, 因为有些数据太大或者不方便放在网上, 所以和代码放在一起是不方便的.

    备注:

    以上的命名最好用中午, 以及连字符-来分割.
