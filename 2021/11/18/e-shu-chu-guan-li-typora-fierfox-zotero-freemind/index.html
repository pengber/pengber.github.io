    背景

    什么是学习?

    上了大学很是迷惑，面临新的科目时，只给一门科目名，其他的都需要自己去寻找，学习资料，学习方法，考核标准，实践标准等，不同于中学的老师给你所有的安排妥当，大学只能自己探索，所以自然而然的思考：

    什么是学习？什么是读书？读书的结果是什么？

    去看了一些书，比如《如何阅读一本书》，《怎样读书》，《读书的艺术》，《我的苦学经验》等方法论的书籍，做了读书笔记笔记，但尝试些许，并没有成功，最关键的原因是：

    1. 坚持不下来。
    2. 打破自己已往的学习习惯，形成新的学习习惯很难。
    3. 没有搞明白知识体系是什么东西。
    人的自身如何塑造?

    于是，转而又思考其他问题：

    人存在的形式是什么？我为什么做不到以上几点，自己何以为自己？自控力怎样获得？记忆力又如何加强？为什么我会在看书时思前想后，注意力又是怎样的一门科学？

    然后又去读了一些书，比如《高效能人士的实践和个人管理法则》，《提高你的记忆力》，《注意力曲线》等等，还找了知乎上的有些回答来看。

    什么叫做会一门技能?

    还有自己的专业，如何评定一个人是否会C++？他学习好的证据是什么？当时他学了《计算机网络》时成绩很高，但是现在一问他知识，他说他忘了。

    那他到底算不算会计算机网络？

    结论

    最终的结论是：

    1. 知识的本质就是记忆，而知识体系的构建可以帮助回忆和记忆。
    2. 人的本质也是记忆，人的各项能力可以自我塑造，但是依然建立在记忆的基础之上，你必须要不断重复自己的目标，自己所处的方位，必须摒弃现实世界混乱信息的干扰，要不忘初心。
    3. 到底会不会某个技能，这个标准也是在你的知识体系下的一个结果。

    所以，我必须建立起自己的知识体系，别人的形式如下

    而我必须在长时间的坚持下形成自己的知识体系。

    博客类型对比

    第三方博客平台

    优点: 没有技术门槛

    缺点:

    1. 网站的内容可能因为网站本身被丢失, 比如博客园整改 gitee人工审核.
    2. 可定制化方案少, 无法按照自己的知识体系进行梳理.

    自己搭建博客例如wordpress

    优点:

    1. 带有后台的网站, 可以管理文章, 评论, 相册, 资源, 用户等.
    2. 可以在线编辑文章和内容.

    缺点:

    1. 需要自己购买域名和服务器.
    2. wordpress采用的文本编辑器是谷腾堡, 如果脱离了wordpress内容很难迁移.
    3. 不能在本地进行创作(可以在本地进行创作后再复制粘贴到网站)
    4. 服务器的部署和维护可能会占用一定精力, 不能专注于内容.
    5. 可迁移性差.

    静态博客

    优点:

    1. 可迁移性高, 可以本地创作, 只是将静态博客作为一个展示窗口, 耦合性低.
    2. 技术难度小(相比于自己建站).
    3. 可以高度定制化
    4. 专注于写作

    缺点:

    1. 静态博客想要实现交互(评论)功能时, 不太方便, 需要使用第三方接口, 关于静态博客的评论系统介绍了一些解决方案.

    我关于静态博客技术方案的需求

    静态博客需求:

    1. 本地md创作的内容经过生成发布到github page
    2. 可以选择不发布某些内容
    3. 可以根据文件目录生成对应的分类
    4. 根据文件本身的创立时间和更新时间更新文件的更新时间
    5. 可以根据md文件的相互本地引用自动替换成对应的url, 比如我引用了本地的D:/xx/xx, 它会自动替换成publish后的url
    6. 可以根据文件名称生成博客标题
    7. 可以映射自己的域名到github
    8. 本地编写完内容后, 推送到github, 自动更新图片为图床链接并生成静态网站更新.
    9. 可以看到文章字数和预估阅读时间.
    10. 访客及流量统计(正反馈)
    11. 评论功能(之前还觉得比较麻烦, 但是看了如果有评论的话正反馈会比较足)
    12. 可以根据目录进行加密(可以自己写工具配合hexo-bolg-encrypt tags进行加密)

    图床比较

    我的图床需求:

    1. 可以一次性上传所有图片不限制数量
    2. 可以下次性下载所有图片不限制数量
    3. 最好不限制图片本身的大小
    图床名称 优点 缺点 备注
    github 1. 稳定
    2. 可以一次性提交所有文件和下载所有文件    		 1. 做图床会扰乱正常的提交记录
    2. 需要连vpn
    gitee 1. 国内不需要连外网
    2. 可以一次性提交所有文件和下载所有文件    		 1. 已经不能作图床了, 开启了防盗链
    2. 仓库刚开始是私有的, 需要公开申请
    3. 国内不稳定因素
    sm.ms 1. 每次最多只能上传10个文件
    七牛云 1. 免费10G
    2. 可以一次性上传所有图片    		 1. 国内访问需要有备案的域名, 也就意味着你自己需要有服务器
    2. 不能一次性下载所有文件
    imgURL 1. 注册账户每月400张
    2. 每次最多上传9张图片

    解决方案

    选择了使用github作图床, 但是不是每复制一次图片就上传, 而是先在使用图片时, 利用typora先复制到本地, 然后定期每月30月底上传一次.

    初始操作

    1. 在本地D:\pic-bed\img执行

      git init
git add .
git commit -m "20220710"

    2. 远程仓库建立pic-bed公共仓库, 注意不要添加README.md

    3. 本地执行

      git remote add origin git@github.com:pengber/pic-bed.git
git branch -M main
git push -u origin main

    之后的操作

    1. 设置typora的图片操作为, 使用时复制到本地路径, 本地路径为D:\pic-bed\img

    2. 每月月底在D:\pic-bed\img将本地未上传的进行更新

      git add .
git commit -m "日期"
git push

    3. 然后利用notepadd++打开D:\pengber.github.io\source\_posts\mdCreating, 替换文件夹内的所有内容中 https://raw.githubusercontent.com/pengber/pic-bed/main/img/ 为 https://raw.githubusercontent.com/pengber/pic-bed/main/img/ 就可以更新本地图片为图床图片了.
